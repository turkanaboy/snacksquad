# U8 Emil interaction review

| Before | After | Why |
| --- | --- | --- |
| A locked feature could read like a dead end. | The pilot gate shows the five measured unlock signals and explains that moderator approval is deliberate. | Members understand why fantasy is unavailable without exposing draft actions early. |
| A draft deadline could be communicated only as backend state. | The draft room keeps status, pick number, active manager, and deadline in one persistent strip. | Time-sensitive state stays visible without timers, pulsing, or distracting motion. |
| Mobile fantasy could add a fifth bottom-navigation destination. | Fantasy remains nested under Contests on mobile while becoming a dedicated desktop item only when enabled. | Navigation preserves the four-item mobile hierarchy and the confirmed product contract. |
| Auto-pick could feel arbitrary. | Managers can build and save an ordered preference queue, while auto-picked history is labeled explicitly. | Automation remains understandable and auditable. |
| Waiver controls could allow an invalid category switch to appear successful. | The UI requires an outgoing roster choice and the database atomically enforces same-category replacement, exclusivity, and the one-Friday limit. | Immediate controls stay simple while authoritative validation prevents misleading state. |

All material findings from this review were resolved in U8.
